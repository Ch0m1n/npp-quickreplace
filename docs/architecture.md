# Architecture

## 목표

NppQuickReplace 0.3 알파는 입력 지연과 데이터 손실 없이 구분 문자·즉시 치환을 제공하고, JSON을 직접 편집하지 않아도 되는 네이티브 규칙 관리 흐름을 제공해요.

## 입력 흐름

```text
SCN_CHARADDED
  -> IME tentative 입력, 전체/문서 상태, 읽기 전용, 다중 선택 확인
  -> 구분 문자 또는 immediate 후보 분류
  -> 버퍼별 캐시에서 파일 확장자 조회
  -> 커서 앞 제한 범위를 SCI_GETTEXTRANGEFULL 한 번으로 읽기
  -> Scintilla 코드 페이지에서 UTF-16으로 변환하고 Unicode 경계 탐색
  -> NFC 정규화 후 RuleStore 해시 인덱스 조회
  -> 결과를 현재 문서 코드 페이지로 안전하게 변환
  -> RAII Undo + 내부 편집 가드에서 SCI_REPLACETARGET
```

표현할 수 없는 문자를 ANSI/DBCS 문서에 넣어야 한다면 치환을 건너뛰고 로그에 경고해요. 플러그인은 문서 인코딩을 임의로 바꾸지 않아요.

## 컴포넌트

### RuleStore

- JSON을 임시 구조로 전부 검증한 뒤 성공할 때만 현재 인덱스와 교체해요.
- 트리거를 Unicode NFC로 정규화하고, 영문 ASCII 대소문자 미구분 인덱스를 별도로 유지해요.
- `immediate` 규칙을 지원하고 더 긴 트리거를 가로채는 접두어 충돌은 거부해요.
- 스키마 버전, 그룹/규칙 ID, activation, 중복, 접두어를 검사하고 경고를 반환해요.
- 64 MiB 파일, 100,000개 규칙, 128 MiB 전체 텍스트 한도를 적용해요.

### ConfigStore

- 기존 JSON의 알 수 없는 필드를 보존하면서 알려진 필드만 갱신해요.
- 고유 임시 파일에 기록하고 `FlushFileBuffers` 후 `ReplaceFileW` 또는 `MoveFileExW`로 교체해요.
- 프로세스 간 이름 있는 mutex로 설정 쓰기를 직렬화해요.
- 규칙 저장 전 시간 스탬프 백업을 만들고 설정된 개수만 남겨요.

### RuleManager

- 일반 Win32 모델리스 창이며 Notepad++에 `NPPM_MODELESSDIALOG`로 등록해요.
- 좌측 목록은 검색·그룹 필터·정렬·다중 선택을 제공해요.
- 우측 편집기는 실행 조건, 확장자, 여러 줄 치환, 설명을 초안으로 편집해요.
- 모든 초안 변경은 RuleStore로 다시 검증하며 저장 전에 자동 백업해요.
- Notepad++ 다크 모드 메시지와 per-monitor DPI 변경을 따라가요.

### Plugin

- 버퍼 ID별 확장자와 문서별 일시정지 상태를 관리해요.
- 파일 저장·이름 변경·활성화 때 캐시를 갱신하고 파일 닫힘 때 상태를 제거해요.
- 설정과 규칙 재로드를 하나의 메뉴 명령으로 묶어요.
- Notepad++ API 경계에서 예외를 잡아 호스트로 전파되지 않게 해요.

## 현재 제한

- 대소문자 미구분은 영문 ASCII에만 적용돼요.
- 붙여넣기 자동 치환은 안전상 비활성 상태예요.
- 다중 선택/커서는 안전 모드에서 건너뛰어요.
- 그룹 자체의 추가·삭제 UI와 CSV/TSV 입출력은 0.4 범위예요.
- 실제 Notepad++ 호스트의 IME·ANSI/DBCS 조합은 수동 검증이 추가로 필요해요.
