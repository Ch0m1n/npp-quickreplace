# Notepad++ 단어 자동 치환 플러그인 기획서

## 1. 문서 정보

- 문서명: Notepad++ 단어 자동 치환 플러그인 기획서
- 가칭 프로젝트명: **QuickReplace for Notepad++**
- 문서 버전: 1.0
- 작성 목적: 미리 등록한 축약어를 입력 즉시 지정된 문구로 치환하여 반복적인 문서 작성 작업을 단축하는 Notepad++ 플러그인의 기능 및 개발 범위를 정의한다.
- 대상 사용자:
  - 게임 데이터 및 패치 노트를 자주 작성하는 사용자
  - 반복되는 전문 용어를 자주 입력하는 번역가
  - 긴 제품명, 기술명, 고유명사를 반복 입력하는 사용자
  - 정해진 문구와 템플릿을 빠르게 삽입하려는 사용자

---

## 2. 프로젝트 개요

### 2.1 핵심 아이디어

사용자가 사전에 등록한 **축약어**와 **치환 문구** 목록을 기반으로, Notepad++ 편집기에 축약어가 입력되면 이를 자동으로 완전한 문구로 바꾼다.

예시 등록값:

```text
ㄱ123 = 대공 방어 사격
```

사용자가 다음과 같이 입력:

```text
ㄱ123 지속시간 30초 -> 40초
```

자동 치환 후:

```text
대공 방어 사격 지속시간 30초 -> 40초
```

### 2.2 해결하려는 문제

- 긴 전문 용어를 반복 입력해야 하는 불편
- 자주 사용하는 문구의 오타 및 표기 불일치
- 번역문이나 패치 노트 작성 시 반복 작업으로 인한 시간 낭비
- 개인 사전이나 외부 프로그램을 별도로 열어 복사·붙여넣기 해야 하는 번거로움
- 일반 자동완성 기능은 후보 선택이 필요하고, 원하는 즉시 치환 방식과 다름

### 2.3 목표

1. 축약어 입력만으로 긴 문구를 즉시 삽입한다.
2. 치환 목록을 사용자가 직접 쉽게 추가·수정·삭제할 수 있게 한다.
3. 일반 텍스트 입력을 방해하지 않도록 오작동을 최소화한다.
4. 파일 형식, 언어, 문서 종류별로 치환 사용 여부를 제어할 수 있게 한다.
5. 설정과 치환 목록을 백업하거나 다른 PC로 옮길 수 있게 한다.
6. 대량의 치환 항목이 등록되어도 입력 지연이 발생하지 않게 한다.

## 3. 주요 용어

| 용어 | 설명 |
|---|---|
| 축약어 | 사용자가 입력하는 짧은 문자열. 예: `ㄱ123` |
| 치환 문구 | 축약어 대신 삽입되는 문자열. 예: `대공 방어 사격` |
| 치환 규칙 | 축약어, 치환 문구, 실행 조건 등을 묶은 하나의 항목 |
| 구분 문자 | 축약어 입력이 끝났음을 판단하는 문자. 예: 공백, Enter, Tab, 마침표 |
| 즉시 치환 | 축약어가 완성되는 순간 바로 치환하는 방식 |
| 구분 문자 치환 | 축약어 뒤에 공백이나 Enter 등을 입력했을 때 치환하는 방식 |
| 스니펫 | 단순 단어를 넘어 여러 줄 문구나 템플릿을 삽입하는 기능 |

## 4. 핵심 사용 시나리오

### 4.1 기본 단어 치환

등록:

```text
ㄱ123 → 대공 방어 사격
```

입력:

```text
ㄱ123 지속시간 30초
```

결과:

```text
대공 방어 사격 지속시간 30초
```

### 4.2 영문 축약어 치환

등록:

```text
wowsaa → 대공 방어 사격
```

입력:

```text
wowsaa 재사용 대기시간 감소
```

결과:

```text
대공 방어 사격 재사용 대기시간 감소
```

### 4.3 문장형 치환

등록:

```text
ㅅㄱㅈ → 수고하셨습니다.
```

입력:

```text
ㅅㄱㅈ
```

결과:

```text
수고하셨습니다.
```

### 4.4 여러 줄 템플릿 삽입

등록:

```text
patchtpl →
변경 전:
변경 후:
변경 사유:
```

입력:

```text
patchtpl
```

결과:

```text
변경 전:
변경 후:
변경 사유:
```

### 4.5 커서 위치가 포함된 템플릿

등록:

```text
balance →
변경 전: ${cursor}
변경 후:
```

치환 후 `${cursor}`가 제거되고 해당 위치에 입력 커서가 놓인다.

## 5. 기능 범위

## 5.1 필수 기능

### 5.1.1 축약어 자동 치환

- 사용자가 입력 중인 현재 문서에서 축약어를 감지한다.
- 등록된 축약어와 일치하면 지정된 치환 문구로 변경한다.
- 기본 실행 시점은 축약어 뒤에 구분 문자가 입력된 순간으로 한다.
- 치환 과정은 Notepad++의 일반 실행 취소 기능으로 되돌릴 수 있어야 한다.
- 한 번의 `Ctrl+Z`로 전체 자동 치환을 취소할 수 있어야 한다.

### 5.1.2 치환 목록 관리

사용자는 플러그인 설정 창에서 다음 작업을 할 수 있어야 한다.

- 새 치환 항목 추가
- 기존 항목 수정
- 항목 삭제
- 항목 활성화 및 비활성화
- 축약어 또는 치환 문구 검색
- 항목 정렬
- 중복 축약어 확인
- 여러 항목 일괄 삭제
- 목록 가져오기 및 내보내기

### 5.1.3 치환 활성화 토글

Notepad++ 메뉴 또는 툴바에서 다음 기능을 제공한다.

```text
플러그인
└─ QuickReplace
   ├─ 자동 치환 활성화
   ├─ 치환 목록 관리
   ├─ 현재 문서에서 일시 중지
   ├─ 마지막 치환 되돌리기
   ├─ 설정
   └─ 정보
```

- 전체 자동 치환 켜기/끄기
- 현재 문서에서만 일시 정지
- 상태 표시줄 또는 메뉴 체크 표시로 활성화 상태 확인

### 5.1.4 치환 실행 조건

각 치환 규칙에 다음 조건을 지정할 수 있어야 한다.

- 공백 입력 시
- Enter 입력 시
- Tab 입력 시
- 문장 부호 입력 시
- 축약어가 완성되는 즉시
- 사용자가 지정한 단축키 입력 시
- 대소문자 구분 여부
- 단어 전체 일치 여부

기본 권장값:

```text
공백, Enter, Tab 입력 시 치환
단어 전체 일치 사용
영문 대소문자 구분 안 함
```

## 5.2 권장 기능

### 5.2.1 수동 치환 단축키

자동 치환이 꺼져 있더라도 사용자가 단축키를 누르면 커서 앞의 축약어를 치환한다.

예시:

```text
기본 단축키: Ctrl + Alt + Space
```

동작:

1. 현재 커서 앞의 단어를 가져온다.
2. 치환 목록에서 일치하는 항목을 찾는다.
3. 일치하면 치환한다.
4. 일치하지 않으면 아무 동작도 하지 않거나 작은 알림을 표시한다.

### 5.2.2 마지막 치환 되돌리기

- 마지막 자동 치환만 별도로 되돌리는 명령을 제공한다.
- 기본적으로 Notepad++의 `Ctrl+Z`와 충돌하지 않게 한다.
- 선택 기능으로 제공하며, 핵심 복구 방식은 Notepad++ 기본 실행 취소를 사용한다.

### 5.2.3 문서 유형별 적용

다음 조건에 따라 자동 치환 적용 여부를 정할 수 있다.

- 파일 확장자
- Notepad++ 언어 모드
- 파일 경로
- 파일명 패턴

예시:

```text
.txt, .md, .po, .xml에서는 활성화
.cpp, .py에서는 비활성화
```

### 5.2.4 그룹 및 프로필

치환 항목을 그룹으로 관리한다.

예시 그룹:

- 공통 문구
- World of Warships
- 영어 번역
- 영상 스크립트
- 프로그래밍
- 개인 메모

각 그룹은 개별 활성화/비활성화가 가능해야 한다.

### 5.2.5 치환 미리보기

목록 관리 창에서 항목을 선택하면 다음을 표시한다.

```text
입력: ㄱ123 지속시간 30초
결과: 대공 방어 사격 지속시간 30초
```

### 5.2.6 충돌 검사

다음과 같은 문제를 저장 전에 알려준다.

- 동일한 축약어가 두 개 이상 등록됨
- 축약어가 비어 있음
- 치환 문구가 비어 있음
- 축약어 앞뒤에 불필요한 공백이 있음
- 축약어가 다른 축약어의 접두어임

예시:

```text
aa
aaa
```

이 경우 즉시 치환 모드에서는 충돌 가능성이 있으므로 경고한다.

## 5.3 확장 기능

초기 버전에서는 제외해도 되지만 향후 추가할 수 있다.

### 5.3.1 변수 치환

```text
${date}       현재 날짜
${time}       현재 시간
${filename}   현재 파일명
${filepath}   현재 파일 전체 경로
${clipboard}  클립보드 내용
${cursor}     치환 후 커서 위치
```

예시:

```text
작성일: ${date}
파일: ${filename}

${cursor}
```

### 5.3.2 여러 커서 위치

```text
함선명: ${1}
티어: ${2}
설명: ${3}
```

Tab을 누를 때마다 `${1}`, `${2}`, `${3}` 위치로 이동한다.

### 5.3.3 정규식 기반 치환

예시:

```text
트리거: hp(\d+)
결과: 내구도 $1
```

입력:

```text
hp50000
```

결과:

```text
내구도 50000
```

정규식은 오작동 가능성이 높으므로 고급 설정으로 분리한다.

### 5.3.4 입력 후보 팝업

사용자가 축약어 일부를 입력하면 일치하는 후보를 보여준다.

예시:

```text
ㄱ1
 ├─ ㄱ123 → 대공 방어 사격
 ├─ ㄱ124 → 군함 수리반
 └─ ㄱ125 → 엔진 가속기
```

### 5.3.5 치환 통계

- 가장 자주 사용한 축약어
- 치환 횟수
- 절약한 예상 타자 수
- 최근 사용 항목

## 6. 상세 동작 규칙

## 6.1 기본 치환 흐름

1. 사용자가 키보드로 문자를 입력한다.
2. 플러그인이 Scintilla 편집기의 문자 입력 이벤트를 감지한다.
3. 입력된 문자가 구분 문자인지 확인한다.
4. 커서 위치를 기준으로 바로 앞의 단어 범위를 계산한다.
5. 해당 단어를 치환 사전에서 조회한다.
6. 일치하는 활성화 규칙이 있으면 원래 축약어를 삭제한다.
7. 지정된 치환 문구를 삽입한다.
8. 사용자가 입력한 구분 문자는 그대로 유지한다.
9. 치환 전체를 하나의 실행 취소 작업으로 묶는다.
10. 필요한 경우 커서 위치 변수를 처리한다.

## 6.2 구분 문자

기본 구분 문자:

- 공백
- Tab
- Enter
- 마침표 `.`
- 쉼표 `,`
- 콜론 `:`
- 세미콜론 `;`
- 느낌표 `!`
- 물음표 `?`
- 닫는 괄호 `)`
- 닫는 대괄호 `]`
- 닫는 중괄호 `}`

사용자 설정에서 구분 문자를 추가하거나 제거할 수 있어야 한다.

### 예시

등록:

```text
aa → 대공 방어 사격
```

입력:

```text
aa,
```

결과:

```text
대공 방어 사격,
```

쉼표는 치환 결과 뒤에 그대로 남아야 한다.

## 6.3 즉시 치환 모드

즉시 치환 모드에서는 마지막 글자가 입력되는 순간 치환한다.

장점:

- 입력이 매우 빠름
- 구분 문자 입력이 필요 없음

단점:

- 더 긴 축약어와 충돌할 수 있음
- 일반 단어 일부가 의도치 않게 치환될 수 있음

따라서 초기 기본값은 **구분 문자 입력 시 치환**으로 한다.

## 6.4 단어 전체 일치

등록:

```text
aa → 대공 방어 사격
```

단어 전체 일치가 켜진 경우:

```text
aa      → 치환
aaaa    → 치환하지 않음
testaa  → 치환하지 않음
aa_test → 설정에 따라 치환하지 않음
```

## 6.5 대소문자 처리

영문 축약어의 대소문자 옵션:

1. 구분함
2. 구분하지 않음
3. 입력 형태를 결과에 반영

초기 버전은 1번과 2번만 지원한다.

## 6.6 한글 입력기 조합 처리

한글 IME 사용 시 조합 중인 문자를 잘못 감지하지 않아야 한다.

예시:

- `ㄱ`
- `가`
- `각`

조합이 완료되기 전에는 치환 검사를 실행하지 않는다.

필수 고려 사항:

- Windows 한글 입력기
- Microsoft IME
- 한/영 전환 직후 입력
- 자음만으로 구성된 축약어
- `ㄱ123`처럼 한글 자모와 숫자가 섞인 축약어

## 6.7 붙여넣기 처리

기본 정책:

- 일반 키 입력만 자동 치환
- 붙여넣은 전체 텍스트는 자동 치환하지 않음

이유:

- 대량 붙여넣기 시 예상치 못한 다수 치환 방지
- 성능 저하 방지
- 원문 훼손 방지

선택 옵션:

```text
붙여넣기 후 마지막 단어만 검사
붙여넣은 텍스트 전체에서 치환
붙여넣기에서는 자동 치환 안 함
```

초기 기본값은 `붙여넣기에서는 자동 치환 안 함`으로 한다.

## 6.8 선택 영역 처리

사용자가 텍스트를 선택한 상태에서 축약어를 입력하면 Notepad++ 기본 동작에 따라 선택 영역이 대체된다.

치환 검사는 새로 입력된 텍스트만 대상으로 한다.

## 6.9 여러 커서 및 다중 선택

Notepad++의 다중 편집 기능을 사용하는 경우:

- 초기 버전에서는 기본 커서 위치만 지원할 수 있다.
- 다중 선택이 감지되면 자동 치환을 건너뛰는 안전 모드를 권장한다.
- 향후 각 커서 위치별 치환 지원을 검토한다.

## 6.10 읽기 전용 문서

읽기 전용 문서에서는 자동 치환을 실행하지 않는다.

## 6.11 자동 저장 및 파일 변경

치환은 일반 입력과 동일하게 문서를 수정된 상태로 만든다.

플러그인이 임의로 파일을 저장해서는 안 된다.

## 9. 기술 설계 방향

## 9.1 개발 언어 후보

### 선택안 A: C++

장점:

- Notepad++ 공식 플러그인 구조와 가장 직접적으로 연동 가능
- Scintilla 메시지 처리에 유리
- 배포 파일을 단일 DLL로 구성하기 쉬움
- 성능과 호환성이 좋음

단점:

- UI와 JSON 처리 구현 난도가 비교적 높음
- 개발 및 디버깅 시간이 길 수 있음

### 선택안 B: C#

장점:

- 설정 창과 목록 관리 UI 구현이 쉬움
- JSON 직렬화 및 데이터 관리가 편리함
- 유지보수가 비교적 쉬움

단점:

- Notepad++ 플러그인 호스팅 방식과 런타임 배포를 신중히 설계해야 함
- Notepad++ 버전 및 .NET 런타임 호환성 고려 필요

### 권장

안정적인 공식 플러그인 배포를 목표로 한다면 **C++ 기반**을 우선 검토한다.

빠른 프로토타입과 개인 사용 목적이라면 C# 또는 PythonScript 플러그인 기반 시제품으로 동작을 먼저 검증한 뒤 C++ 플러그인으로 옮기는 방법도 가능하다.

## 9.2 Notepad++ 연동 지점

플러그인은 다음 이벤트 또는 기능과 연동해야 한다.

- 문자 입력 알림
- 커서 위치 조회
- 현재 문서 텍스트 조회
- 범위 텍스트 교체
- 실행 취소 그룹 시작 및 종료
- 현재 파일 경로 조회
- 현재 언어 모드 조회
- 문서 전환 알림
- Notepad++ 종료 알림
- 플러그인 메뉴 명령 등록

Scintilla 관련 핵심 처리 개념:

- 현재 위치 확인
- 커서 앞 단어 범위 계산
- 대상 범위 텍스트 읽기
- 대상 범위 교체
- 실행 취소 작업 묶기
- 타깃 범위 설정 후 문자열 교체

## 9.3 치환 검색 자료구조

치환 항목 수에 따른 권장 방식:

### 소규모 목록

- `unordered_map<string, ReplacementRule>`
- 수백~수천 개 항목까지 충분히 빠름

### 접두어 검색 또는 즉시 치환

- Trie 자료구조
- 자동완성 후보 표시
- 축약어 접두어 충돌 검사

초기 버전 권장:

```text
정확 일치 검색: 해시맵
목록 검색 및 정렬: 별도 벡터
```

## 9.4 문자열 인코딩

Notepad++와 Scintilla는 문서 인코딩 및 내부 UTF-8 처리 상태를 고려해야 한다.

필수 지원:

- 한글
- 영문
- 숫자
- 일본어
- 특수 문자
- 이모지
- 여러 줄 문자열

권장 내부 저장 방식:

```text
UTF-8
```

JSON 파일도 UTF-8로 저장한다.

## 9.5 실행 취소 처리

자동 치환은 다음 전체 과정이 하나의 실행 취소 단계가 되어야 한다.

```text
축약어 삭제
+
치환 문구 삽입
+
구분 문자 유지
```

사용자가 `Ctrl+Z`를 한 번 누르면 자동 치환 직전 상태로 돌아가야 한다.

## 9.6 재귀 치환 방지

치환 결과 안에 다른 축약어가 포함되어 있어도 즉시 다시 치환해서는 안 된다.

예시:

```text
a → aa
aa → 전체 문구
```

`a`를 치환한 결과 `aa`가 생성되어도 같은 이벤트 안에서 연속 치환하지 않는다.

방지 방법:

- 플러그인 내부 편집 중 플래그 설정
- 치환으로 발생한 수정 이벤트는 무시
- 사용자 입력 이벤트에서만 검사

## 9.7 성능 목표

- 10,000개의 치환 항목이 등록되어도 일반 입력 지연이 체감되지 않아야 한다.
- 한 번의 검색은 평균적으로 1ms 이내를 목표로 한다.
- 매 입력마다 전체 문서를 검색하지 않는다.
- 커서 주변의 필요한 범위만 읽는다.
- 목록 파일은 시작 시 메모리에 로드한다.
- 목록 변경 시에만 검색 인덱스를 다시 생성한다.

## 10. 예외 및 충돌 처리

## 10.1 중복 축약어

동일한 축약어가 여러 개 존재할 경우 저장을 막는 것을 기본 정책으로 한다.

대안:

- 그룹 우선순위
- 파일 형식별 우선순위
- 사용자 선택 팝업

초기 버전에서는 복잡성을 줄이기 위해 동일 축약어 중복 등록을 허용하지 않는다.

## 10.2 접두어 충돌

예시:

```text
ㄱ1
ㄱ12
ㄱ123
```

구분 문자 치환 방식에서는 큰 문제가 없다.

즉시 치환 방식에서는 `ㄱ1` 입력 순간 먼저 치환될 수 있으므로 다음 중 하나를 적용한다.

- 즉시 치환 규칙끼리 접두어 충돌이 있으면 저장 경고
- 가장 긴 일치를 기다리는 지연 시간 사용
- 즉시 치환에서는 접두어 관계 규칙 금지

초기 권장 정책:

```text
즉시 치환 규칙 간 접두어 충돌 금지
```

## 10.3 치환 결과가 빈 문자열

단어 삭제 기능으로 사용할 수 있으나 실수 가능성이 크다.

초기 버전에서는 빈 치환 문구 저장을 금지한다.

향후 `삭제 규칙` 타입으로 별도 지원할 수 있다.

## 10.4 매우 긴 치환 문구

- 최대 길이를 설정하거나 경고를 표시한다.
- 기본 권장 최대 길이: 100,000자
- 대용량 템플릿은 별도 파일 스니펫 기능으로 확장 가능

## 10.5 설정 파일 손상

동작:

1. 현재 설정 파일 읽기 실패
2. 자동 백업 중 가장 최근의 정상 파일 검색
3. 복구 여부 안내
4. 복구할 수 없으면 빈 기본 목록으로 시작
5. 손상 파일은 덮어쓰지 않고 별도 이름으로 보존

예시:

```text
replacements.corrupted.20260710.json
```

## 10.6 외부 편집 충돌

사용자가 `replacements.json`을 외부 편집기로 수정할 수 있다.

선택 정책:

- 설정 창을 열 때 파일 변경 여부 확인
- 변경되었으면 다시 불러오기 안내
- 자동 감시 기능은 향후 버전에서 지원

## 15. 보안 및 안정성 요구사항

- 문서 내용을 외부로 전송하지 않는다.
- 인터넷 연결 없이 모든 기능이 동작해야 한다.
- 플러그인 오류가 Notepad++ 전체 종료로 이어지지 않게 예외를 처리한다.
- 설정 저장 전 임시 파일을 만든 뒤 원자적으로 교체한다.
- 목록 저장 전 자동 백업을 생성한다.
- 치환 목록 파일에 실행 코드가 들어가더라도 실행하지 않는다.
- 정규식 기능이 추가될 경우 처리 시간 제한을 고려한다.
- 현재 문서가 수정 가능한 상태인지 확인한 뒤 치환한다.

권장 저장 방식:

```text
1. replacements.json.tmp에 새 내용 저장
2. 저장 성공 및 JSON 유효성 검사
3. 기존 replacements.json을 백업
4. tmp 파일을 replacements.json으로 교체
```

## 16. 사용성 원칙

1. 사용자가 입력하지 않은 문장을 임의로 수정하지 않는다.
2. 기본 설정은 오작동보다 안전성을 우선한다.
3. 자동 치환은 즉시 되돌릴 수 있어야 한다.
4. 치환이 발생한 사실을 지나치게 방해되는 팝업으로 알리지 않는다.
5. 목록 편집은 엑셀과 유사하게 빠르게 처리할 수 있어야 한다.
6. 대량 항목을 파일로 편집하고 가져올 수 있어야 한다.
7. 기능을 끄는 방법이 항상 명확해야 한다.
8. 현재 문서에서만 일시 정지할 수 있어야 한다.

## 19. 완료 조건

다음 조건을 모두 만족하면 1차 개발 목표가 완료된 것으로 본다.

- 사용자가 설정 창에서 `ㄱ123 → 대공 방어 사격` 규칙을 등록할 수 있다.
- 편집기에서 `ㄱ123`을 입력한 뒤 공백을 누르면 자동 치환된다.
- 공백은 치환 결과 뒤에 정상적으로 남는다.
- 한글 IME 조합 중에는 잘못 치환되지 않는다.
- 한 번의 `Ctrl+Z`로 자동 치환 전체를 취소할 수 있다.
- 플러그인을 끄면 자동 치환이 발생하지 않는다.
- Notepad++를 재시작해도 치환 목록이 유지된다.
- 치환 항목 10,000개 기준으로 입력 지연이 체감되지 않는다.
- 설정 파일이 손상되어도 Notepad++가 종료되지 않는다.
- 문서 내용이 외부로 전송되지 않는다.

## 21. 프로젝트 요약

본 플러그인은 Notepad++에서 반복적으로 사용하는 긴 단어, 전문 용어, 문장 또는 템플릿을 짧은 축약어로 입력하고 자동으로 치환하는 생산성 도구다.

가장 중요한 1차 목표는 다음 동작을 빠르고 안정적으로 구현하는 것이다.

```text
등록: ㄱ123 → 대공 방어 사격
입력: ㄱ123 지속시간 30초 -> 40초
결과: 대공 방어 사격 지속시간 30초 -> 40초
```

초기 버전은 안전한 구분 문자 기반 치환, 한글 IME 대응, 실행 취소, JSON 기반 목록 저장에 집중한다. 이후 그룹, 파일 형식별 적용, 여러 줄 스니펫, 커서 변수, 자동완성 후보 등의 기능을 단계적으로 확장한다.
